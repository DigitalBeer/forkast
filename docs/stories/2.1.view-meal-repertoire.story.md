## Story
**As a** user,  
**I want** to see a dedicated page listing all the meals I have saved,  
**so that** I can view my entire repertoire in one place.

## Acceptance Criteria
1. A "My Meals" page exists at `/meals`
2. The page displays all meals from Supabase (for logged-in users) or local storage (for anonymous users)
3. If no meals exist, show a friendly message with an "Add Your First Meal" button
4. Each meal card displays:
   - Meal name
   - Image (if available)
   - Tags/categories
   - Last prepared date
   - Quick actions (edit, delete, add to plan)
5. The list is searchable by meal name and tags
6. The list is sortable by:
   - Name (A-Z, Z-A)
   - Last prepared (newest, oldest)
   - Most used
7. Responsive design works on mobile and desktop

## Dev Notes
### Technical Implementation
- Use Supabase `select` with RLS for authenticated users
- Implement local storage adapter for anonymous users
- Use `useEffect` and `useState` for data management
- Consider using `react-query` for data fetching and caching
- Implement virtualized list for better performance with large datasets

### UI/UX Considerations
- Show skeleton loaders while data is loading
- Add subtle animations for better perceived performance
- Ensure keyboard navigation works for all interactive elements
- Add ARIA labels for accessibility

### Testing
- Test with both authenticated and anonymous users
- Verify all interactive elements work on touch and mouse devices
- Test with various screen sizes
- Verify performance with large numbers of meals
